COMMUNITY COMPLAINT TRACKER

Project Description

Community Complaint Tracker is a lightweight complaint management system
built using Python, Streamlit, and Pandas. It allows citizens to report
community issues such as potholes, garbage, water leakage, electricity
failures, and public safety concerns. Complaint records are stored in a
simple pipe-separated text file (complaints.txt), making the project
easy to understand and suitable for learning Python file handling.

Features

-   Register new complaints
-   Automatic complaint categorization using keyword matching
-   Browse and search complaints
-   Filter by category and status
-   Download complaints as CSV
-   Dashboard with complaint statistics and charts

Objectives

-   Demonstrate Python file handling
-   Perform input validation
-   Use dictionaries for categorization
-   Build a simple web application using Streamlit
-   Visualize complaint statistics

Technologies Used

-   Python
-   Streamlit
-   Pandas
-   Text File Handling
-   Dictionaries
-   Exception Handling

Algorithms Used

1.  Keyword Matching
2.  Sequential Complaint ID Generation
3.  Dictionary Counting
4.  Linear Search

Project Structure

Community Complaint Tracker/ | |– complaint_tracker.py |– complaints.txt

Installation

1.  Install Python.
2.  Install packages: python -m pip install streamlit pandas
3.  Run: python -m streamlit run complaint_tracker.py

Data Storage

All complaints are stored in complaints.txt.
